# Market Conditions スコアの問題点分析

> 本ドキュメントは現行の Market Conditions 算出式を、代表的なマーケットセンチメント指標の設計思想と比較し、「なぜスコアが想定より高く出るのか」を構造的に検討したものです。

---

## 1. 問題の要約

現行スコアは以下の構造的特性により、**実際の市場環境よりもブル寄りの数値を恒常的に出力する傾向がある**。

主要因は3つに集約される。

1. **コンポーネントの大半が「比率そのまま」をスコアにしており、変換関数がない** — 正常な上昇相場では % above SMA 系が軒並み60〜80%に張り付くため、加重合計すると容易にスコア60〜70台に達する。
2. **下方リスクを検知するコンポーネントが根本的に不足している** — CNN Fear & Greed Indexなどが標準的に組み込むPut/Call比率、Junk Bond Demand、Safe Haven Demand、New Highs/Lows比率がいずれも欠落。
3. **VIXの扱いが軽すぎ、かつ変換関数が甘い** — 重みがわずか5%で、VIX 20（市場が明らかに神経質な水準）でもスコア68を返す。

---

## 2. 代表的な指標との比較

### 2.1 CNN Fear & Greed Index

CNN Fear & Greed Index は7つの等重みコンポーネントの平均（各0〜100）で構成される。

各コンポーネントの特徴は以下の通り。

- **Market Momentum**: S&P 500 と 125日移動平均の乖離。現行実装にはこの「インデックスレベルのモメンタム」に相当するものがない。
- **Stock Price Strength**: 52週高値 vs 52週安値の銘柄数比率。現行の pct_2w_high は2週間高値のみで52週高値/安値の比率を見ていない。
- **Stock Price Breadth**: McClellan Volume Summation Index。Advance/Decline に出来高を加味した指標で、単純な「SMA上の銘柄%」よりはるかに情報量が多い。
- **Put/Call Options**: CBOE Put/Call比率。オプション市場を通じた恐怖の直接的な計測。**現行実装に該当コンポーネントなし。**
- **Market Volatility**: VIX。CNN ではこれに全体の1/7（≒14.3%）の重みを割り当てている。現行の5%は約1/3の影響力しかない。
- **Safe Haven Demand**: 株式 vs 国債の20日リターン差。リスク回避の動きを直接測る指標。**現行実装に該当コンポーネントなし。**
- **Junk Bond Demand**: ジャンク債と投資適格債のスプレッド。信用リスクへのセンチメント。**現行実装に該当コンポーネントなし。**

CNN方式と比較した現行実装の最大の差異は、**7指標のうち3指標（Put/Call、Safe Haven、Junk Bond）に相当するコンポーネントが存在しない**点にある。この3つはいずれも「市場参加者の恐怖」を直接測る指標であり、欠落することで下方バイアスの検知能力が大幅に低下する。

### 2.2 % Above SMA 系指標の業界標準的な解釈

StockChartsやBarchartなど主要チャートサービスでは、% above 200-day SMA に以下のような閾値を適用するのが標準的。

- **70%超**: 買われすぎ（overbought）圏。ブル相場でもこの水準が長期間続くのは異常。
- **50%前後**: 中立。ブルとベアの境目。
- **30%以下**: 売られすぎ（oversold）圏。

つまり % above SMA200 が 70% のとき、これは「ブリッシュ」ではなく「既にかなり過熱している」と読むのが標準。しかし現行実装では70%をそのままスコア70として扱い、「Positive」ラベル（≥60）に分類する。これは方向性として正しいがインデックスの精度としては不十分で、「正常なブル相場」と「過熱したブル相場」の区別がつかない。

さらに重要な点として、StockChartsのPercent Above Moving Averageの解説では50%ラインのクロスが特に重要とされている。200日SMAの場合、50%を下回るとダウントレンド入りのシグナルと見なされる。しかし現行実装はこの閾値の概念を持たず、「65%なら65点」という線形スケーリングしか行っていない。

### 2.3 McClellan Oscillator / Summation Index

McClellan Oscillator は (19日EMA − 39日EMA) of (Advances − Declines) で算出される。これが示すのは「ブレッドスの変化率」であり、現行の % above SMA 系が示す「ブレッドスの水準」とは根本的に異なる。

水準ベースの指標だけでは、ゆっくり悪化しているマーケットの変化を捉えられない。2024年後半のように、S&P 500が新高値をつけながらA/Dラインが下落するようなダイバージェンスが発生した場合、現行のスコアはおそらくBullish圏を維持するが、McClellan系を組み込んでいれば悪化を検知できる。

### 2.4 TRIN (Arms Index)

TRIN = (Advancing Issues / Declining Issues) / (Advancing Volume / Declining Volume) で算出され、1.0を中心に上下する。NYSEのウォール表示に常時掲示されるほど重要な短期指標。

TRINが特に有用な点は、「出来高を加味したブレッドス」を測る点にある。現行の % above SMA 系は純粋に価格ベースであり、出来高情報がまったく入っていない。Volume Accumulation スキャンのように個別銘柄レベルでは出来高を見ているにもかかわらず、マーケット全体のコンディション判定では出来高を無視しているのは矛盾する。

### 2.5 New Highs / New Lows 比率

52週新高値銘柄数 / (新高値 + 新安値) は、0〜100でオシレーターとして使える。Hindenburg Omenなどの暴落警告シグナルも新高値/新安値データに依存している。

現行の pct_2w_high（2週間高値の比率）は、これの大幅に弱い代替にすぎず、重みも5%しかない。52週高値/安値の比率は市場のレジーム（ブル/ベア）判定に最も信頼性の高い指標の一つだが、現行実装には含まれていない。

## 3. コンポーネント別の問題分析

### 3.1 SMA系（全体重み 72%）の構造問題

pct_above_sma10 (12%) + pct_above_sma20 (12%) + pct_above_sma50 (14%) + pct_above_sma200 (14%) + pct_sma20_gt_sma50 (10%) + pct_sma50_gt_sma200 (10%) で合計72%。

**問題1: 時間軸の重複が大きい。** SMA10 > SMA20 > SMA50 > SMA200 は入れ子構造になっている。close > SMA200 の銘柄は高確率で close > SMA50 でもある。SMA20 > SMA50 の銘柄群は SMA50 > SMA200 とも高い相関を持つ。結果として6つの指標が事実上3つ分程度の情報量しか持たず、にもかかわらず72%の重みを占めている。正常なブル相場ではこれらが軒並み60〜80%に揃うため、他のコンポーネントがいくら低くてもスコアが60を下回りにくい。

**問題2: 生の比率をそのまま使っている。** 業界標準では、% above SMA は非線形変換（パーセンタイルランク、Z-score、または閾値ベースのスコアリング）を経て0-100にマッピングする。例えばCNN Fear & Greed の各コンポーネントは「過去2年間のレンジにおける現在値の位置」でスコアリングされる。これにより、「平常時は50付近、極端な時だけ0や100に近づく」というキャリブレーションが実現する。現行実装は % above SMA = 65% ならスコア65という直接マッピングなので、平常時でもスコアが高止まりする。

**問題3: ラグのある指標に過大な重みがかかっている。** SMA50, SMA200, SMA20>SMA50, SMA50>SMA200 は変化が非常に遅い指標。マーケットが天井を打っても、これらが50%を割るまでに何週間もかかる。その間ずっとスコアを下支えしてしまう。

### 3.2 Return系（全体重み 18%）の問題

pct_positive_1m (10%) + pct_positive_ytd (8%)。

**問題: pct_positive_ytd はカレンダー依存で不安定。** 年初来リターンは1月と12月で意味がまったく異なる。1月時点では数日分のノイズ、12月時点では1年分の蓄積。これを年間通じて同じ重みで使うのはスコアの安定性を損なう。

さらに、pct_positive_1m と pct_positive_ytd はどちらも「方向だけ」を見ている（正か負か）。リターンの大きさを考慮しないため、「かろうじてプラスの銘柄が大量にある」と「大幅プラスの銘柄が大量にある」が同じスコアになる。

### 3.3 pct_2w_high（重み 5%）の問題

10日ローリング高値到達比率。2週間という短さでは市場のレジーム判定にはほぼ貢献しない。52週新高値/新安値比率に置き換えるか、少なくとも補完すべき。

### 3.4 VIX スコア（重み 5%）の問題

```
vix_score = clamp(100 - max(VIX - 12, 0) × 4, 0, 100)
```

この変換関数の問題点は以下の通り。

**VIX 12以下が全て100点になる。** VIX の歴史的な中央値は概ね17〜20付近。VIX 12は歴史的にかなり低く、Extreme Greed圏に近い。これを100点（最高ブル）にするのは、CNN Fear & Greedが低VIXを「Greed」として減点する方向とまったく逆。低VIXは「安心感」ではなく「自己満足（complacency）」のシグナルとして扱うのが業界の常識。

**減衰が遅すぎる。** VIX 20でスコア68、VIX 25でスコア48、VIX 37で初めて0。VIX 20〜25は既にマーケットが不安定な水準であり、この時点でスコア50前後はブル寄りすぎる。

**重みが軽すぎる。** CNN Fear & Greedでは7指標の1つ（≒14.3%）。現行は5%。VIXが急騰しても全体スコアへのインパクトが微小。

## 4. ETFユニバースモードの追加問題

デフォルトの `calculation_mode = etf` は、market_condition_etf_universe のETF群でコンポーネントを計算する。

**問題: サバイバーシップバイアスとセレクションバイアス。** ETFユニバースの銘柄選定が市場全体のブレッドスを正しく反映しているかどうかが重要。セクターETFやファクターETFが含まれている場合、それらは市場全体より分散が小さく、SMA上にある比率が高くなりやすい。S&P 500構成銘柄全体や、NYSEの全銘柄と比較すると、ETFユニバースは本質的にブル方向にバイアスがかかる可能性がある。

---

## 5. 改善の方向性

以下は問題の深刻度順に並べた改善案。

### 5.1 コンポーネントのリスコアリング（最重要）

全コンポーネントに対して、生の比率をそのまま使うのではなく「過去N日間（例: 252日 = 1年）のパーセンタイル位置」に変換する。これにより以下の効果が得られる。

- 平常時のスコアが自動的に50付近にキャリブレーションされる
- 異常に高い/低い値のみが極端なスコアを出す
- マーケットレジームの変化に対する感度が大幅に向上する

```python
# 概念例
raw_value = pct_above_sma50  # 例: 72%
score = percentile_rank(raw_value, history_252d)  # 過去1年で72%がどの位置か
```

CNN Fear & Greedが各コンポーネントを「過去2年のレンジにおける位置」でスコアリングしているのは、まさにこの原理。

### 5.2 SMA系の重み削減と冗長性の解消

現行72%のSMA系重みを40〜50%程度に削減し、以下のように整理する。

- **短期ブレッドス**: pct_above_sma20（代表1本に統合）
- **中期ブレッドス**: pct_above_sma50
- **長期ブレッドス**: pct_above_sma200
- **トレンド構造**: pct_sma50_gt_sma200（1本に統合）

pct_above_sma10 と pct_sma20_gt_sma50 は冗長として削除候補。

### 5.3 欠落コンポーネントの追加

空いた重み枠に以下を追加する（ETFモードでも取得可能なものを優先）。

- **New Highs / New Lows 比率**: 52週新高値銘柄数 / (新高値 + 新安値)。市場のレジーム判定に最も信頼性が高い。ETFモードで取得不可の場合はactive_symbols モードで計算。
- **Advance/Decline Volume Ratio**（簡易McClellan的指標）: 上昇出来高 / 下落出来高の短期EMA。TRINの考え方を単純化したもの。
- **Safe Haven Demand 的指標**: SPY vs TLTの20日リターン差など。ETFベースで容易に計算可能。
- **Put/Call比率**: 外部データ（CBOE）が必要だが、取得できるなら追加価値が大きい。

### 5.4 VIX変換関数の修正

基準点をVIX 17付近（歴史的中央値）に設定し、そこからの上下偏差でスコアリングする。

```python
# 改善案の概念例
vix_median = 17.0
vix_deviation = (VIX - vix_median) / vix_median  # 中央値からの相対偏差
vix_score = clamp(50 - vix_deviation * 80, 0, 100)
# VIX 12 → スコア約74（低VIXだが100ではない）
# VIX 17 → スコア50（中立）
# VIX 25 → スコア約12（ベア圏）
# VIX 37 → スコア0
```

これに加えて、VIXの重みを5%から10〜15%に引き上げる。

### 5.5 ラベル閾値の調整

リスコアリング（5.1）を実施すれば、現行の閾値（80/60/40/20）はそのまま使える可能性がある。ただし、リスコアリング導入前の暫定策としては、閾値を引き上げる（例: 85/70/50/30）ことで「甘い」判定を緩和できる。

### 5.6 ダイバージェンス検出の追加（将来的な拡張）

スコア自体とは別に、「インデックス価格が上昇しているのにブレッドスが悪化している」ダイバージェンスを検出するレイヤーを追加する。2024年末のMagnificent 7集中相場のような環境で、スコアが見かけ上高いまま実態が悪化するケースを捕捉できる。

## 6. 具体的なスコアシミュレーション

典型的な「正常なブル相場」での各指標の想定値と、現行 vs 改善案でのスコア差を示す。

### 正常なブル相場の想定値

```
pct_above_sma10   = 62%
pct_above_sma20   = 58%
pct_above_sma50   = 70%
pct_above_sma200  = 75%
pct_sma20_gt_sma50 = 72%
pct_sma50_gt_sma200 = 68%
pct_positive_1m   = 60%
pct_positive_ytd  = 65%
pct_2w_high       = 35%
VIX               = 18
```

**現行スコア計算:**
```
62×0.12 + 58×0.12 + 70×0.14 + 75×0.14 + 72×0.10 + 68×0.10 +
60×0.10 + 65×0.08 + 35×0.05 + vix_score(18)×0.05

vix_score(18) = 100 - (18-12)×4 = 76

= 7.44 + 6.96 + 9.80 + 10.50 + 7.20 + 6.80 + 6.00 + 5.20 + 1.75 + 3.80
= 65.45 → "Positive"
```

これは一見妥当に見えるが、問題はこれが「正常なブル相場」つまり平常状態である点。平常状態で65（Positive寄り）という結果は、スコアの上方余地が35しかなく、情報量が低い。比較的悪い局面を検出するためには、平常時が50付近に収まるべき。

**パーセンタイル変換後の概念的スコア:**
仮にすべてのコンポーネントを「過去1年のパーセンタイル」に変換すると、正常なブル相場は概ね50〜60パーセンタイル付近に位置し、合成スコアは55前後の「Neutral寄りのPositive」になる。これなら上方にも下方にもスコアの変動余地があり、マーケット環境の変化に対してより敏感に反応する。

## 7. 結論

現行のMarket Conditionsスコアが「甘い」最大の理由は、**コンポーネントの生値を直接スコアとして使っていること**と、**恐怖/リスク回避系のコンポーネントが不足していること**の2点に帰着する。

パーセンタイルベースのリスコアリングを導入するだけで、SMA系の冗長性問題と上方バイアスの問題を同時に大幅に改善できる。コンポーネント追加（New Highs/Lows、Safe Haven Demand等）とVIX変換の修正を組み合わせれば、CNN Fear & Greedに近い精度と感度を持つスコアに進化させることが可能。
