## 前提：重複回避ルール

前回の分析で判明した包含関係を設計に反映します。

- `21EMA scan (V2)` と `Pullback Quality` は同一プリセット内で併用しない（ほぼサブセット関係）
- `VCS` と `VCS 52 High` は同一プリセット内で併用しない（完全包含関係）
- `Pullback Quality` と `Reclaim` は時系列的に相補関係にあるため、セットとして使う場合は「状態→トリガー」の流れを意識する

---

## Preset 1：Leader Breakout（主導株ブレイクアウト）

**方針：** 相対強度・総合力ともに最上位の銘柄が、ベースを締めて発射台にいる状態を捕捉する。

**スキャンセット：** `97 Club` + `VCS 52 High` + `Three Weeks Tight`

**閾値：** 3スキャン中 **2以上** で重複

**なぜこの組み合わせか。** 97 Clubは `hybrid_score >= 90` かつ `raw_rs21 >= 97` かつ `trend_base` で、RS・ファンダメンタル・業種の総合力が最上位の銘柄だけを通す。VCS 52 Highは `vcs >= 60` かつ `rs21 > 60` かつ `dist_from_52w_high >= -15` で、高値圏でボラティリティが収縮中であることを確認する。Three Weeks Tightは `three_weeks_tight == True` かつ `trend_base` かつ `vcs >= 50` で、週足レベルでの締まりを見る。

97 Clubが「誰が主導株か」を、VCS 52 Highが「収縮しているか」を、Three Weeks Tightが「週足でも固まっているか」を別々の角度から確認している。97 Clubの条件にはVCSが含まれないし、VCS 52 HighにはHybridスコアが含まれないので、重複ではなく相互補完になっている。

VCS単体（`vcs >= 60` + `rs21 > 60`）は入れていない。VCS 52 Highが完全にVCSを包含するため、同一プリセット内に入れると1つのスキャンを2回カウントすることになる。

---

## Preset 2：Orderly Pullback（構造的押し目）

**方針：** すでに確認済みの強い銘柄が、秩序ある調整で21EMA付近に戻り、まだリクレイムしていない段階を捕捉する。

**スキャンセット：** `Pullback Quality` + `Volume Accumulation` + `RS Acceleration`

**閾値：** 3スキャン中 **2以上** で重複

**なぜこの組み合わせか。** Pullback Qualityは10条件を持つ厳格なスキャンで、21EMA scanのサブセットとして設計されている。`trend_base`、MAスロープの上昇、`atr_21ema_zone` が -1.25〜0.25（21EMA近辺〜やや下）、`drawdown_from_20d_high_pct` が 3〜15%、そして決定的なのが `volume_ma5_to_ma20_ratio <= 0.85`（出来高冷却）。これだけで「秩序ある押し目」のフィルターとして機能する。

Volume Accumulationは `ud_volume_ratio >= 1.5`（上昇日出来高が下落日出来高の1.5倍以上）かつ `rel_volume >= 1.0` かつ当日陽線で、需給の方向性を確認する。Pullback Qualityが「押し目の質」を見ているのに対し、Volume Accumulationは「買い手がまだ優勢か」を見ており、観点が異なる。

RS Accelerationは `rs21 > rs63` かつ `rs21 >= 70` かつ `trend_base` で、短期RSが中期RSを上回り加速中であることを確認する。押し目にいるのに相対強度が加速しているということは、その銘柄の調整が市場全体の調整より浅い＝相対的に強い証拠になる。

21EMA scan（V2）は入れていない。Pullback QualityとほぼV2の厳格版であり、同一プリセット内での併用は重複カウントになるため。

---

## Preset 3：Reclaim Trigger（リクレイム・トリガー）

**方針：** 押し目からの復帰の「瞬間」を捕捉する。Preset 2が「構え」ならPreset 3は「トリガー」。

**スキャンセット：** `Reclaim` + `Pocket Pivot` + **`Fundamental Demand`（新規提案）**

**閾値：** 3スキャン中 **2以上** で重複

**なぜこの組み合わせか。** Reclaimは `close_crossed_above_ema21`（当日21EMA上抜け）かつ `min_atr_21ema_zone_5d <= -0.25`（直近5日で一度は21EMA下にいた証拠）かつ `volume_ratio_20d >= 1.10`（出来高増加）という、リクレイムの瞬間そのものを定義するスキャン。

Pocket Pivotは `close > sma50` かつ当日の出来高が過去10日の最大出来高を超えるという、機関投資家の買い行動を日次で検出する。Reclaimと同日にPocket Pivotが重なれば「出来高を伴った21EMA上抜け」であり、トリガーの信頼度が上がる。

**新規スキャン提案：Fundamental Demand**

現行スキャンセットに「ファンダメンタルの質で単独フィルターするスキャン」が存在しない。97 Clubの `hybrid_score >= 90` にファンダメンタルは含まれるが、これはRS・業種との合成スコアであり、ファンダメンタル単体の強さを直接見ていない。アノテーションリストに `High Est. EPS Growth` があるが、スキャン（重複カウント対象）にはなっていない。

```python
# Scan Spec: Fundamental Demand
# Canonical name: Fundamental Demand
# UI display name: Fund Demand

matched = bool(
    row.get("fundamental_score", 0.0) >= 70.0
    and _raw_rs(row, 21) >= 60.0
    and row.get("rel_volume", 0.0) >= 1.0
    and row.get("daily_change_pct", 0.0) > 0.0
    and row.get("trend_base", False)
)
```

**Required Inputs:**

| Field | Producer | Default | Scan use |
|---|---|---|---|
| `fundamental_score` | `src/scoring/fundamental.py` | `0.0` | `>= 70.0` |
| `raw_rs21` / `rs21` | `src/scoring/rs.py` | `nan` | `>= 60.0` |
| `rel_volume` | `src/indicators/core.py` | `0.0` | `>= 1.0` |
| `daily_change_pct` | `src/indicators/core.py` | `0.0` | `> 0.0` |
| `trend_base` | `src/indicators/core.py` | `False` | `True` |

**Config Dependencies:**

| Config key | Default | Used as |
|---|---|---|
| `scan.fund_demand_fundamental_min` | `70.0` | lower bound for `fundamental_score` |
| `scan.fund_demand_rs21_min` | `60.0` | 下限RS21 |
| `scan.fund_demand_rel_vol_min` | `1.0` | 下限相対出来高 |

**設計意図：** EPSと売上成長がユニバース内で上位30%以内で、RSもそこそこあり、当日の出来高が平均以上で陽線。「ファンダメンタルが良い銘柄に、当日需要が来た」ことを捉える。Reclaimの「テクニカル的な復帰トリガー」とFundamental Demandの「ファンダ+需要の確認」が重複すれば、質の高いトリガーになる。

## Preset 4：Momentum Surge（爆発的モメンタムの持続性確認）

**方針：** すでに動き出した銘柄の中から、一発屋ではなく持続的な主導株を識別する。

**スキャンセット：** `4% bullish` + `Momentum 97` + `PP Count` + **`Sustained Leadership`（新規提案）**

**閾値：** 4スキャン中 **2以上** で重複

**なぜこの組み合わせか。** 4% bullishは `daily_change_pct >= 4` かつ `rel_volume >= 1.0` かつ `from_open_pct > 0` かつ `raw_rs21 > 60` で、当日の爆発的な動きを検出する。Momentum 97は `weekly_return_rank >= 97` かつ `quarterly_return_rank >= 85` かつ `trend_base` で、週次・四半期ともにクロスセクショナルに最上位であることを確認する。この2つは名前が似ているが測定軸が全く異なる（当日の動き vs 期間リターンの順位）。

PP Countは `pp_count_30d > 3` かつ `trend_base` で、過去30日に4回以上のPocket Pivot＝継続的な機関需要を確認する。4% bullishの「今日の爆発」がPP Countの「最近ずっと買われている」と重なれば、一過性ではないと判断できる。

**新規スキャン提案：Sustained Leadership**

現行スキャンには「複数の時間軸でRSが一貫して強い」ことを単独で確認するスキャンがない。RS Accelerationは `rs21 > rs63` で加速を見ているが、絶対水準での持続的リーダーシップを捉えるものではない。

```python
# Scan Spec: Sustained Leadership
# Canonical name: Sustained Leadership
# UI display name: RS Leader

rs21 = _raw_rs(row, 21)
rs63 = row.get("rs63", float("nan"))
rs126 = row.get("rs126", float("nan"))

matched = bool(
    pd.notna(rs21) and float(rs21) >= 80.0
    and pd.notna(rs63) and float(rs63) >= 70.0
    and pd.notna(rs126) and float(rs126) >= 60.0
    and row.get("trend_base", False)
)
```

**Required Inputs:**

| Field | Producer | Default | Scan use |
|---|---|---|---|
| `raw_rs21` / `rs21` | `src/scoring/rs.py` | `nan` | `>= 80.0` |
| `rs63` | `src/scoring/rs.py` | `nan` | `>= 70.0` |
| `rs126` | `src/scoring/rs.py` | `nan` | `>= 60.0` |
| `trend_base` | `src/indicators/core.py` | `False` | `True` |

**Config Dependencies:**

| Config key | Default | Used as |
|---|---|---|
| `scan.sustained_rs21_min` | `80.0` | 短期RS下限 |
| `scan.sustained_rs63_min` | `70.0` | 中期RS下限 |
| `scan.sustained_rs126_min` | `60.0` | 長期RS下限 |

**設計意図：** 1か月・3か月・6か月のすべてで市場を上回り続けている銘柄。短期RSの閾値が最も高く、長期に向かって緩くなる設計は「最近さらに加速している持続的リーダー」を捉える。RS Accelerationが「加速の傾き」を見るのに対し、Sustained Leadershipは「全期間での絶対的な強さ」を見ており、重複ではなく相互補完。

## Preset 5：Early Cycle Recovery（サイクル初期の反転候補）

**方針：** 弱気局面の終盤や底打ち後に、最初にベース形成し始める次サイクルのリーダー候補を早期発見する。

**スキャンセット：** `VCS 52 Low` + `Volume Accumulation` + **`Trend Reversal Setup`（新規提案）**

**閾値：** 3スキャン中 **2以上** で重複

**なぜこの組み合わせか。** VCS 52 Lowは `vcs >= 60` かつ `rs21 > 60` かつ `dist_from_52w_low <= 25` で、安値圏だがボラティリティが収縮し始めている銘柄を捉える。ここにVolume Accumulationが重なれば、安値圏にもかかわらず上昇日の出来高が下落日を上回っている＝静かに買い集められている兆候。

**新規スキャン提案：Trend Reversal Setup**

現行スキャンはほぼすべて `trend_base == True`（close > sma50 かつ wma10_weekly > wma30_weekly）を要求する。これは上昇トレンド銘柄に特化した設計であり、底打ち初期の銘柄は構造的に排除される。VCS 52 Lowだけが `trend_base` を要求しないが、「50SMAを上抜けたばかり」「200SMAとの関係が改善し始めた」といった反転初期の構造を見るスキャンがない。

```python
# Scan Spec: Trend Reversal Setup
# Canonical name: Trend Reversal Setup
# UI display name: Reversal Setup

matched = bool(
    row.get("close", 0.0) > row.get("sma50", float("inf"))
    and row.get("sma50", float("inf")) <= row.get("sma200", float("inf"))
    and row.get("sma50_slope_10d_pct", float("nan")) > 0.0
    and row.get("dist_from_52w_low", float("nan")) <= 40.0
    and row.get("dist_from_52w_high", float("nan")) >= -40.0
    and _raw_rs(row, 21) >= 50.0
    and row.get("pp_count_30d", 0) >= 1
)
```

**Required Inputs:**

| Field | Producer | Default | Scan use |
|---|---|---|---|
| `close` | latest price | `0.0` | `> sma50` |
| `sma50` | `src/indicators/core.py` | `inf` | 価格比較 + sma200との関係 |
| `sma200` | `src/indicators/core.py` | `inf` | `sma50 <= sma200`（まだデッドクロス圏内） |
| `sma50_slope_10d_pct` | `src/indicators/core.py` | `nan` | `> 0.0`（50SMAが上向き始めた） |
| `dist_from_52w_low` | `src/indicators/core.py` | `nan` | `<= 40.0`（安値から遠すぎない） |
| `dist_from_52w_high` | `src/indicators/core.py` | `nan` | `>= -40.0`（高値から落ちすぎていない） |
| `raw_rs21` / `rs21` | `src/scoring/rs.py` | `nan` | `>= 50.0` |
| `pp_count_30d` | `src/indicators/core.py` | `0` | `>= 1` |

**Config Dependencies:**

| Config key | Default | Used as |
|---|---|---|
| `scan.reversal_dist_52w_low_max` | `40.0` | 52週安値からの最大距離 |
| `scan.reversal_dist_52w_high_min` | `-40.0` | 52週高値からの最小距離 |
| `scan.reversal_rs21_min` | `50.0` | RS下限（中立以上） |

**設計意図の核心は `sma50 <= sma200` という条件。** これは50SMAがまだ200SMAの下にある＝長期デッドクロスが続いている状態を意味する。通常のスキャンが `trend_base`（close > sma50 かつ週足MA上昇）でフィルターするのに対し、このスキャンは「長期的にはまだ弱いが、50SMAが上向き始め、価格が50SMAを上抜け、需要の兆候がある」という反転初期の構造を狙う。VCS 52 Lowの「安値圏で収縮」とTrend Reversal Setupの「構造が改善し始めた」が重なれば、次サイクルの主導株候補をかなり早い段階で発見できる。

## 全体設計の俯瞰

| Preset | 市場環境 | 性質 | スキャン数 | 閾値 |
|---|---|---|---|---|
| 1. Leader Breakout | 強気 | 発射台の検出 | 3 | 2+ |
| 2. Orderly Pullback | 強気 | 状態の確認 | 3 | 2+ |
| 3. Reclaim Trigger | 強気 | トリガーの捕捉 | 3 | 2+ |
| 4. Momentum Surge | 強気 | 持続性の判定 | 4 | 2+ |
| 5. Early Cycle Recovery | 弱気→転換期 | 早期発見 | 3 | 2+ |

新規スキャン提案は3つです。

1. **Fundamental Demand** — ファンダメンタル上位＋当日需要。現行にファンダ単独スキャンがない穴を埋める。
2. **Sustained Leadership** — 全時間軸でRS上位の持続的リーダー。97 Clubとは条件構造が異なる（Hybridスコアではなく生RSの多期間確認）。
3. **Trend Reversal Setup** — `sma50 <= sma200` の反転初期構造。現行の `trend_base` 依存のスキャン群では構造的に捕捉できない領域をカバーする。

いずれも既存のインジケータースタック（`src/indicators/core.py` と `src/scoring/` 配下）で算出済みのフィールドのみを使っており、新規インジケーターの追加は不要です。`sma200` は `03_INDICATORS_AND_SCORING.md` で既に `rolling_mean(close, 200)` として実装されていることを確認済みです。
